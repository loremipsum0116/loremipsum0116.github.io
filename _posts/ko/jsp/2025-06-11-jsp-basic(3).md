## JSP 완전 정복 1탄: 동적인 웹 페이지, 대체 어떻게 만드는 건데?

오늘은 웹 개발의 재미있는 첫걸음, 바로 **JSP(Java Server Pages)**에 대해 이야기해 보겠습니다. "웹 페이지는 알겠는데, 동적인 건 또 뭐야?" "JSP는 왜 쓰는 거지?" 같은 궁금증이 많으실 텐데요, 하나씩 차근차근 파헤쳐 봅시다.

---

### 🌟 JSP, 너는 누구냐? (feat. 동적인 웹 페이지)

간단히 말해서 JSP는 **동적인 웹 페이지를 만들 때 사용하는 자바 기반의 웹 프로그래밍 기술**이에요. 여기서 '동적'이라는 말에 주목해야 합니다!

* **정적인 웹 페이지**: 우리가 흔히 보는 `.html`, `.css`, `.js` 파일이나 이미지처럼 **저장된 파일 그대로 브라우저에 전송되어 출력**되는 페이지를 말해요. 클라이언트(사용자)가 100번 요청해도, 누가 요청해도, 언제 요청해도 **항상 똑같은 모습**을 보여주죠. 마치 미리 인쇄된 전단지 같이 말이죠.

* **동적인 웹 페이지**: 반면에 동적인 웹 페이지는 똑같은 주소로 접속해도 **요청할 때마다 내용이 달라질 수 있는 페이지**예요. 서버가 클라이언트의 요청을 받아서 실시간으로 데이터를 가공하고, 그 결과에 따라 새로운 HTML 문서를 만들어서 브라우저에 보내줍니다. 예를 들어, 여러분이 로그인하면 '마이페이지'에 여러분의 이름과 주문 내역이 뜨고, 친구가 로그인하면 친구의 이름과 주문 내역이 뜨는 것! 이게 바로 동적인 웹 페이지의 마법입니다. 은행 잔고 조회, 실시간 뉴스 피드, 쇼핑몰 장바구니 등이 모두 동적인 페이지의 예시죠.

JSP는 바로 이렇게 **사용자의 요청에 따라 내용이 바뀌는 웹 페이지를 자바 언어를 사용해서 서버에서 똑똑하게 만들어 브라우저로 전송**해주는 역할을 합니다.

---

### 💡 서블릿 vs. JSP: 누가 더 화면 만들기에 편할까?

JSP를 배우다 보면 '서블릿'이라는 친구를 만나는데요. 둘 다 자바 기반으로 동적인 웹 페이지를 만들 수 있는데, 무슨 차이가 있을까요?

#### **서블릿 (Servlet)**

서블릿은 "웹을 다룰 수 있는 자바 클래스"라고 생각하면 돼요. 기본적으로 순수 **자바 코드**로 구성되어 있죠. `HttpServlet`이라는 클래스를 상속받아서 웹에서 필요한 기능들(요청 처리, 응답 생성 등)을 다루게 됩니다.

하지만 서블릿으로 HTML 페이지를 만드는 건 생각보다 꽤 불편한데요. HTML 태그 하나하나를 `out.println()` 같은 자바 코드로 직접 생성하고 조합해야 하거든요. 마치 그림을 그릴 때 점 하나하나를 찍어가며 그리는 느낌? 코드가 너무 길어지고 복잡해지기 쉬워서 HTML 구조를 한눈에 파악하기가 어려워집니다.

#### **JSP (Java Server Pages)**

이런 서블릿의 불편함을 해소하기 위해 등장한 것이 바로 JSP입니다! JSP는 **HTML을 기본 골격으로 하고, 그 안에 필요한 부분만 자바 코드를 쏙쏙 삽입하는 형태**를 가지고 있어요. 마치 HTML 문서에 '자바 코드를 실행해줘!'라고 마법의 주문을 거는 것과 같이 훨씬 직관적이고 편리합니다.

| 구분             | 서블릿                                        | JSP                                            |
| :--------------- | :-------------------------------------------- | :--------------------------------------------- |
| **HTML 생성 방식** | 자바 코드 안에서 `out.println()` 등으로 전체 HTML 페이지를 생성 (매우 번거로움) | HTML 코드 안에서 `<% ... %>` 같은 스크립트 형태로 자바 코드를 필요한 부분에만 추가 (훨씬 직관적) |
| **변수 초기화** | 변수 선언 및 초기화를 직접 코드로 선행해야 함 | 웹 컨테이너에서 제공하는 `request`, `response` 같은 **내장 객체**를 즉시 사용 가능 |
| **주요 용도** | 주로 클라이언트의 요청을 받아 비즈니스 **로직을 처리하는 '컨트롤러'** 역할 | 주로 처리된 결과를 사용자에게 보여주는 **'뷰(View/화면)'** 역할 |

실제 웹 개발에서는 이 둘을 같이 사용하는 경우가 많습니다. 서블릿이 클라이언트의 요청을 받아 핵심 로직을 처리하고, JSP는 그 처리 결과를 예쁘게 포장해서 사용자에게 보여주는 역할을 분담하는 거죠. (흔히 MVC 패턴이라고 부르는데, 이건 나중에 더 자세히 다뤄보겠습니다!)

### SP 파일, 이렇게 생겼어! (기본 구조 파헤치기)

JSP 파일은 겉보기에는 HTML 파일과 매우 비슷하게 생겼어요. 하지만 그 안에 숨겨진 자바 코드를 실행시키는 특별한 '요소'들이 있습니다.

```jsp
<%@ page language="java" contentType="text/html; charset=UTF-8"
    pageEncoding="UTF-8"%> <%-- 💡 1. 지시어: JSP 페이지의 설정 정보 --%>
<!DOCTYPE html>
<html>
<head>
<meta charset="UTF-8">
<title>처음 만들어보는 JSP</title>
</head>
<body>
    <%! // 💡 2-1. 스크립트 요소 - 선언부 (멤버 변수/메서드 선언)
        String str1 = "JSP";
        public String getGreeting() {
            return "안녕하세요!";
        }
    %>

    <h1><%= getGreeting() %>, <%= str1 %>!</h1> <%-- 💡 2-2. 스크립트 요소 - 표현식 (값 출력) --%>
    <p>
        <% // 💡 2-3. 스크립트 요소 - 스크립트릿 (자바 코드 로직)
            String str2 = "반갑습니다.";
            out.println(str2 + " 웹 페이지입니다. 함께 공부해요!");
        %>
    </p>
</body>
</html>
```

위 코드를 보면 크게 세 가지 유형의 특별한 요소들이 보여요.

#### 1. 지시어 (`<%@ ... %>`)

* **역할**: JSP 페이지가 서블릿 코드로 변환될 때 필요한 정보를 JSP 엔진(WAS 내부)에 알려주는 역할을 해요. JSP 페이지의 '설정 파일' 같은 거죠. '디렉티브'라고도 불려요.
* **종류**:
    * **`page`**: 가장 많이 사용하는 지시어로, 현재 JSP 페이지에 대한 전반적인 정보를 설정해요. 어떤 언어를 쓸 건지(`language`), 어떤 형식으로 웹 페이지를 만들 건지(`contentType`), 인코딩은 뭘로 할 건지(`pageEncoding`), 필요한 자바 클래스는 뭔지(`import`) 등등 중요한 설정을 담당합니다.
    * **`include`**: 다른 외부 파일을 현재 JSP 파일 안에 '포함'시킬 때 사용해요. 웹 페이지의 공통 부분(헤더, 푸터)을 재활용할 때 아주 유용합니다.
    * **`taglib`**: EL(Expression Language)에서 자바 클래스의 메서드를 호출하거나 JSTL(JSP Standard Tag Library) 같은 외부 라이브러리 태그를 사용하기 위해 선언합니다.

#### 2. 스크립트 요소 (HTML 중간에 자바 코드를 삽입할 때 사용)

JSP의 가장 큰 특징이자 핵심! HTML 코드 안에 자바 코드를 '삽입'하는 문법이에요.

* **선언부 (`<%! ... %>`)**:
    * **역할**: JSP 페이지 전체에서 공유하고 사용할 **멤버 변수**나 **메서드**를 선언하는 영역이에요.
    * **특징**: 서블릿으로 변환될 때 `_jspService()`라는 핵심 메서드 **바깥쪽**에 위치하게 돼요. 그래서 이 변수나 메서드들은 해당 JSP 페이지(서블릿)의 '글로벌'한 요소가 됩니다.

* **표현식 (`<%= ... %>`)**:
    * **역할**: 어떤 변수의 값이나 메서드 호출 결과처럼 **하나의 값을 웹 브라우저 화면에 바로 출력**하고 싶을 때 사용해요.
    * **특징**: 마치 자바에서 `System.out.println()`이나 `out.print()`를 쓰는 것과 똑같은 효과를 냅니다. 값을 직접적으로 '표현'하는 거죠.

* **스크립트릿 (`<% ... %>`)**:
    * **역할**: JSP 페이지가 요청을 받았을 때 실행해야 할 **자바 코드 로직**을 작성하는 영역이에요. 반복문, 조건문, 변수 선언 및 연산 등 대부분의 자바 코드는 여기에 작성해요.
    * **특징**: 서블릿으로 변환될 때 `_jspService()` 메서드 **안쪽**에 그대로 들어가게 돼요. 그래서 이곳에 선언된 변수는 이 스크립트릿 내부에서만 유효한 **지역 변수**가 됩니다. (참고로 스크립트릿 안에서는 새로운 메서드를 선언할 수 없고, 선언부에 정의된 메서드만 호출할 수 있어요!)

### `page` 지시어, 속성별로 파고들기!

가장 중요하고 자주 쓰이는 `page` 지시어의 주요 속성들을 좀 더 자세히 알아볼까요?

* **`language="java"` (기본값: `java`)**:
    * 이 JSP 페이지에서 사용할 스크립팅 언어를 지정해요. 기본적으로 `java`를 사용하니 크게 신경 쓸 일은 없겠지만, 다른 언어를 사용한다면 변경할 수 있어요.

* **`contentType="text/html; charset=UTF-8"`**:
    * 이 페이지가 생성할 문서의 MIME 타입(종류)과 문자 인코딩을 지정해요. 대부분의 웹 페이지는 `text/html` 타입이고, 한글 깨짐 방지를 위해 `charset=UTF-8`을 사용하는 것이 필수적입니다!

* **`pageEncoding="UTF-8"`**:
    * JSP 소스 코드 파일 자체의 인코딩 방식을 지정해요. 역시 한글 깨짐 방지를 위해 `UTF-8`로 설정하는 것이 좋습니다.

* **`import="java.util.List, java.io.IOException"`**:
    * JSP 페이지에서 외부 자바 패키지나 클래스를 사용하려면 자바처럼 `import` 문으로 가져와야 해요. 필요한 클래스가 있다면 이 속성에 지정하면 됩니다.

* **`session="true"` (기본값: `true`)**:
    * 이 JSP 페이지에서 웹 세션(`session` 내장 객체)을 사용할지 여부를 지정해요. 보통은 `true`로 두고 사용합니다.

* **`errorPage="에러페이지경로.jsp"`**:
    * 이 JSP 페이지에서 자바 코드 실행 중 **예외(오류)가 발생했을 때, 사용자에게 보여줄 에러 페이지의 경로**를 지정해요.

* **`isErrorPage="false"` (기본값: `false`)**:
    * 이 JSP 페이지가 **다른 페이지에서 발생한 에러를 처리하는 전용 페이지인지 여부**를 지정해요. `true`로 설정하면 `exception`이라는 내장 객체를 사용하여 발생한 예외 정보를 얻을 수 있답니다. (위에서 에러 처리 방법 2에서 설명한 그 친구!)

### 🚨 에러 페이지, 예쁘게 처리하고 싶다면? (`errorPage`, `isErrorPage`)

방금 설명한 `errorPage`와 `isErrorPage` 속성은 사용자 경험과 보안에 아주 중요한 역할을 해요.

만약 JSP 페이지에서 에러가 났는데 아무런 처리를 하지 않으면, 보통은 컴퓨터 코드와 알 수 없는 메시지들이 잔뜩 적힌 '500 내부 서버 오류' 페이지가 뜹니다. 상상만 해도 당황스럽죠? 사용자 입장에선 "이 사이트, 뭔가 이상한데?" 하고 바로 나가버릴 거예요. 게다가 코드 일부가 노출될 수도 있으니 보안에도 취약하죠.

이런 상황을 막기 위한 해결책은 크게 두 가지예요!

1.  **`try-catch` 블록으로 코드 안에서 직접 처리**:
    * 자바 개발하듯이, 예외가 발생할 가능성이 있는 코드 부분을 `try { ... } catch (Exception e) { ... }` 블록으로 감싸서 직접 예외를 잡고 처리하는 방법이에요.
    * `request.getParameter("age")`처럼 사용자 입력 값을 숫자로 변환할 때 오류가 나거나, 없는 파일에 접근하려 할 때처럼 특정 코드에서만 예외가 발생할 것이 확실한 경우에 유용해요. 이 페이지 내에서 바로 해결하고 싶을 때 쓰면 좋겠죠!

2.  **`page` 지시어의 `errorPage`, `isErrorPage` 속성 활용**:
    * 이 방법은 에러가 나면 아예 **다른 페이지로 사용자를 이동시켜서 거기서 에러를 보여주는 방식**이에요.
    * **단계:**
        1.  **에러를 받을 페이지 만들기**: `errorPage.jsp` 같은 파일을 만들고, 여기에 예쁜 디자인과 함께 `isErrorPage="true"`라고 선언해요. 이 페이지에서는 `exception`이라는 특별한 내장 객체를 통해 어떤 에러가 났는지, 메시지는 무엇인지 등을 알아낼 수 있어요.
        2.  **에러가 날 수 있는 페이지에 설정**: `mainPage.jsp`처럼 에러가 날 가능성이 있는 페이지의 `page` 지시어에 `errorPage="errorPage.jsp"`처럼 에러 페이지의 경로를 지정해 줍니다.
    * 이렇게 하면 `mainPage.jsp`에서 에러가 터져도 사용자에게 못생긴 오류 페이지 대신, 미리 디자인해 둔 `errorPage.jsp`를 보여줄 수 있으니 훨씬 전문적이고 친절하게 느껴지겠죠? 사이트의 신뢰도도 높아지고요!

오늘은 JSP가 무엇인지, 왜 필요한지부터 시작해서 정적인 웹 페이지와 동적인 웹 페이지의 차이, 서블릿과의 관계, 그리고 JSP 파일의 기본적인 구조와 핵심 지시어들을 자세히 살펴봤어요. 특히 `page` 지시어의 다양한 속성과 에러 처리 방법은 실제 개발에서 정말 많이 사용되니 꼭 기억해 두시면 좋겠습니다!
